Tom [Contact Tracing Coordinator]: And it can’t be limited to just the case. All the contacts need the same structure. We do follow-up on them and if they develop into cases we need to upgrade them, but without breaking the link to the original case they were exposed to. So we need easy linking, not an afterthought.

Raj [IT/SecOps Lead]: On the investigator side, do we store your details? I assume we should capture investigator name, how to contact you, and maybe office address.

Maya [Field Epidemiologist]: Yes, we keep getting bounced around when folks don’t know who owns a record. If the investigator who first interviewed the case is on leave, someone else can pick up the thread if they can find who to call. Name, email, phone, maybe team assignment.

Denise [State Health Dept Leadership]: Also organizations. The hospital or clinic that reported the case, the lab that tested, employer sites. Name, location, contact details for those. When we have to brief leadership we need to be specific: which facility, where, how many linked individuals.

Elena [Laboratory Interface Specialist]: On locations, can we be more granular than just street address? We get situations like a single campus with multiple buildings. You want to log a building, a floor, maybe a room, a store number. And sometimes GPS coordinates help, like for a farm or a mobile pop-up site.

Tom [Contact Tracing Coordinator]: Yeah, and we need to grab other identifying details too. Like location type, office vs warehouse vs classroom, because it changes the risk profiles. And if there’s a unit number, we need that, or we lose track of which room had the exposure.

Maya [Field Epidemiologist]: Travel history is big too. We end up scribbling in margins. If someone was in Miami, then flew to Houston, and then took a bus, we want to capture the timeline in a structured way. Even if we don’t want to get into flight databases right now, we should at least record what the person tells us: routes, dates, where they stayed, overlaps with others.

Denise [State Health Dept Leadership]: And the suspected agent. Pathogen or toxin, or unknown at first and then updated. If it’s anthrax or ricin, that’s a different posture than norovirus. So agent info should be part of the record and show up in analysis.

Raj [IT/SecOps Lead]: Please don’t say ricin out loud. Joking. But point taken. We need a slot for agent data that can be populated as we learn more.

Tom [Contact Tracing Coordinator]: On contacts and exposures, can we assume the system can tie each exposed entity back to the index case? Like, I don’t want a free-floating contact without a link. Every exposure record should say exposed to who or what and when.

Maya [Field Epidemiologist]: And not just people. Sometimes an exposure is a place or a shift, like a lunch room on 5th floor. But yeah, if it’s a contact, it should clearly be tied to a case, and that relationship should be visible later when we review the links.

Elena [Laboratory Interface Specialist]: By links, you mean seeing connections among cases, exposures and the other stuff so you can reason about sources, right?

Tom [Contact Tracing Coordinator]: Exactly. We’ve been doing that in whiteboards. The system should keep those traceable links. If we can trace back to a common gathering or the same batch of something, we should see it.

Denise [State Health Dept Leadership]: Are you saying you want analytics? Because we don’t have a data scientist on call for each event.

Tom [Contact Tracing Coordinator]: No, no. I’m saying the relationships have to be in the records. If later someone wants to analyze, they can. The links must be traceable. At a minimum, we need to know who is connected to whom, and via which exposure type.

Maya [Field Epidemiologist]: And classifying some things as controls during an investigation, like a facility that had no exposure, to compare. We need to mark that status so we can filter them out of certain counts.

Raj [IT/SecOps Lead]: So a flag to classify an entity as a control in the context of a given investigation. That’s doable if we model entity roles per event.
