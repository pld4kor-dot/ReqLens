Features (summary only; details elsewhere)
- Color search engine: find paints by name/number/standard color value (RGB). Use 3rd party DB for paint data + search index [dependency].
- Visual color chooser ('graphical picker'). UI patterns in whitepaper, but spec needs to state capability.
- Image-based match ('color sample matcher' using uploaded image). Server processes image; returns candidates. [Ops note: size limits TBD].
- Translator: map old paint numbers + collections to new numbers + collections. Migration complexities included (adds, retirements). If a color is retired, show nearest alternatives.
- Closest-colors finder: determine colors nearest to a given color; algorithm assumption = RGB distance acceptable (document assumption).
- User color palette: session-based. Default users (non-admin) can use all functionality except admin; their data is session-only, no password-protected storage, expires with session end.
- Admin console: manage paints, collections, and admin users. Admin roles L1/L2/L3 with permissions: L1 = add + create L1; L2 = add/update + create up to L2; L3 = add/update/delete + create any level.
- Error utility: users can report errors. Automatic log sending to vendor (ColorKast) on errors; preserve UX (non-intrusive).
