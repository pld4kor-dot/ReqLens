All,

Summary of field mapping and a few design decisions:

- MSEL Summary fields to display: Title, Venue, Planned Start (exercise time), Planned Duration, Key Event flag, Source App (Sync Matrix 1.0 or EMT 1.0), Source ID, Actual Start (if recorded), Variance (computed as actual vs planned using exercise time). Each item includes a link target for deep linking.

- Variance colors will use the DHS palette. We can add a small text label (Ahead / On Time / Behind) in the legend to reinforce meaning.

- Two clocks on the top bar (local time and exercise time). The timeline’s now indicator aligns to the exercise clock.

- Display adjustments: allow hide/show of specific columns in the right table and let users define time segment boundaries (e.g., Morning = 0800–1200). Timeline granularity options: 5, 15, 30 minutes. We’re not saving presets in v1.

- Auto-refresh at 60 seconds by default. Pause/Resume implemented as a toggle; when paused, polling stops and a Paused ribbon shows. On resume, we fetch current data and align views to the exercise time.

- Availability: plan updates for off-hours. We’ll include a short note about maintenance windows and target availability.

- Change history: if the source apps expose change logs for fields we display, we’ll surface those in a History view. If not, we’ll present a history based on differences detected over time in the data the wallboard ingests (including timestamps and, where provided by sources, the user who made the change). In-app view only—no export in v1.

- Security: the wallboard will prompt for sign-in if the session expires. Deep links won’t bypass source app security.

Open items:

- Exact font sizes/contrast for projector readability—test Wednesday.
- Whether the proximity window for the lower list is fixed +/- 30 min or user-adjustable in v1 (leaning fixed for now).

Regards,
Devon
