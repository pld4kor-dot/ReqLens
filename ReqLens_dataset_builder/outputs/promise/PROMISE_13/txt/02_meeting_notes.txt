  - IBM W3 intranet only. No external exposure. Availability tied to W3 (target 95% of W3 uptime). Avoid advertising 24/7.
  - Authentication: IBM intranet username/password. No automatic login. No cookies storing credentials (no saved login).
  - Authorized CCR users only (include approved non‑US CCR staff). Access group config req’d. HRIS sync? [Jason to confirm]
  - Platform: IBM-issued Windows machines (Windows 2000+). Older workstations must function.
  - Reminder: Browser password saving discouraged; app should not encourage it. (Note: policy doc link [?])

  - Include regions outside US (datasets). Ensure capability to ingest global data fields. (Localization out of scope now.)
  - Access for approved non-US CCR staff. Same auth flow on W3.

– Reporting/Exports (Brio)
  - One-click to generate a report once selection made. One-click to export. No extra confirmation screens.
  - Exports provided via Brio module. Buttons in UI produce spreadsheet files:
    • Excel (.xls)
    • Lotus 1-2-3 (.123) [legacy—still required in some regions]
  - Button labels should be explicit. Keyboard access to export buttons required. No buried menus.
  - Reminder: Data exports reflect latest weekly refresh. (Consider showing last refresh date/time on dashboard? [discussion; not mandatory])
