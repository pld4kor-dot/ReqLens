Hi,

+1 to all. Two ops caveats:

- Uptime: We’re targeting “always available” in practical terms. Please budget infra for ~99.9% availability. If we’re down, it better be a rare blip (think 999 successes out of 1000 checks). Agents will be using this 24/7, literally.

- Load: Plan to handle 1000 concurrent users at launch (tops of hour will spike). Expect 5x growth within a year if recruitment goes as planned. QA should simulate this — Owen, can you plan a load test with bursts around the sync window?

- Preferences: We’ve had too many support tickets where after a crash, an app forgets settings. Can we guarantee that user preferences are retained even if the device restarts, and restored completely when the app comes back up?

- Security: Only licensed/registered agents get in. No assistants piggybacking on someone else’s login. Require authentication and then check authorization for sensitive actions (e.g., listing edits).

- Data hygiene: Please validate inputs thoroughly. We’ve seen negative sale prices, dates in the wrong century, etc. Don’t let trash through to MLS or our office system.

Quick tech notes (typed on the bus, sorry for typos):

- Platforms: Confirming v1 on Windows CE and Palm OS. We’re following native UI conventions: standard menus, hardware/soft buttons, no custom widgets. The look will be professional and we’ll stick to your color scheme + fonts within what the OS supports.

- Performance: With current prototype against the staging MLS feed, search is returning results in ~12–18s in good network, ~24–28s in weak spots, so still under the 30s ceiling. CMA generation is hovering around 45–55s after comp selection, which meets the under 60s target. We’ll keep optimizing.

- Offline: We cache saved searches (criteria + last results snapshot), generated CMAs, and appointment lists. So if the device goes offline, users can view those.

- Notifications: After the hourly sync with the office system completes, we surface “New appointments” if found. Also, when calls are placed from the app to buyer/seller, we can recognize the callback/response and notify the agent.

- Images: We’re tuning thumbnail size to preserve clarity in results while keeping memory usage reasonable. The detail view shows a clearer image.

- Input validation: We’re blocking negative numbers, enforcing date formats, and required fields on listing updates/creation before any submission. This prevents bad data hitting MLS/office endpoints.

- Install: Installer is a single tap package; after install, first run prompts for credentials. No printed guide expected. We’ll do a small in-app nudge. We’d like QA to measure what % of untrained folks complete it on their own (target is 90%+).

- Concurrency/load: We’ll coordinate with QA for a 1000 simultaneous user test and will profile to ensure headroom for the 5x growth. Also watching top-of-hour spikes.

- One caveat: Hourly is fixed because of the office system. A few folks asked for “instant” pulls; not possible in v1.
