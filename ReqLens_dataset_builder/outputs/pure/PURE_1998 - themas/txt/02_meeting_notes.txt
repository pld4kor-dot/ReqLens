— Additional detail bullets —

- External Interface Overview:
  - thermostat input: current temperature + desired setpoint only.
  - outputs to equipment: H/C ON/OFF request signals.
  - no guarantee of execution confirmation from hardware.
  - all traffic through designated interfaces; no direct device manipulation.
- Utilization: max number of units enforced; a waiting queue for excess requests; semantics LIFO. [Rules in §3.1.4.*]
- Events/Alarms: log alarms and operational events. Future reports rely on event database.
- Reporting overview (brief per agenda):
  - Two kinds: historical (12 months) vs monthly statistical summary.
  - Operator chooses which.
  - Output as plain text files; operator picks path/name. [No UI mockups here]
  - Don’t go deep into file schemas; plain ASCII is enough at req level.
- SRS compliance: Prepared per ANSI/IEEE 830-1984. Sections as above.
- References: Include technical references as part of the spec. Public standards are generally available via libraries. Copies of non-public refs via Barry.
- Precedence: If a ref conflicts with SRS, SRS governs, except SoW overrides SRS.

— More notes —

- Monitor Temperature: primary function — read T and TSET; validate; produce status; compute LO/UO and LT/UT for alarms/decisions.
- Determine Utilization: enforce cap; maintain ON/OFF software status; queue LIFO; re-check thermostat need when capacity frees.
- Initialize System: read init file; assign thermostat unique IDs; turn all units OFF to known state before reading values.
- System Reports: produce ASCII file reports; operator selects path/name; modes: 12-month operational history vs monthly statistical.
- External Interfaces identification (overview level): thermostat inputs (T, TSET), unit outputs (H/C ON/OFF Request), supervisor console (human operator). No detailed vendor specs in SRS.
