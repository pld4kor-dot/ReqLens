Hi Ben,

Adding a few PoS/cohort/report points I noted while prepping curriculum lists:

- Program of Study creation/edit: create a PoS with program name and pick the required classes (clinical and non-clinical). Sometimes we rename a program or change the required list—editing should be straightforward.

- Class modeling: when we add new classes, mark them as either non-clinical or clinical. For non-clinical, capture lecture room requirements + instructor needs. For clinical, include clinical site needs and separate fields for lecture instructor needs vs clinical lab instructor needs.

- Cohorts: when adding a cohort, identify it by start month/year (e.g., Sep 2026). Tie the cohort to ONE PoS only. For each class in the sequence, set which quarter it will be offered. We occasionally insert a new class or clinical section midstream.

- Reports/printables we rely on: cohort snapshot (students, PoS, sequence, progress, completion timeline); clinical site summary per quarter (site info + labs, dates/times); clinical lab roster (class name/number, lab section id, list of enrolled students, instructor, days/times, location); individual student summary (name, ID, admission date, classes, credits, GPA, cohort); system log (printable); planning report by quarter (classes, number of sections, number of labs, and room types needed) across all programs/cohorts.

- Directory: please store email + phone for staff, students, lecture instructors, clinical lab instructors, and clinical site admins. This helps notifications and quick outreach.

Let me know if you need examples — I have spreadsheets with sequences and room type notes.
